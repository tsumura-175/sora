# はじまりの家そら デザインシステム

## 基本方針

- 淡い青、白、罫線、写真、余白で「静かで安心できる」印象を保つ。
- 見出しは和文セリフ体、本文は和文サンセリフ体、英字ラベルと番号は英字書体を使う。
- 新規ページで独自の見出し・カード・背景表現を増やさず、既存コンポーネントを優先する。

## 正となるファイル

- 編集元は `src/`。ルート直下のHTML、`css/`、`js/`、`images/` はビルド生成物であり直接編集しない。
- 共通ナビゲーションと連絡先は `src/data/site.json`。
- 下層ページのメタ情報、KV、関連ページ、アンカー情報は `src/pages/<page>/page.json`。
- ページ固有の本文は `src/pages/<page>/main.html`。

## 下層ページ

下層ページは、必要な要素だけを次の順に使う。

1. `{{pageHero}}` — KV、英字ラベル、パンくず
2. 筆線のdivider（必要なページのみ）
3. `{{pageAnchors}}` — 長いページのみ。KV直下のアンカーとPC左追従ナビを同じ設定から生成する
4. 本文セクション
5. `{{pageRelated}}` — 関連性の高い3ページ
6. 共通お問い合わせ、フッター

料金・法人情報のような短い参照ページには、アンカーと左追従ナビを付けない。

## 見出し・余白

- 英字ラベル: `en-sub reveal-sub`
- 本文セクション見出し: `section-title`
- 下層ページ本文: `about-section` を基本にし、情報表・料金表は用途別のページ固有classを使う。
- 色、書体、余白、角丸は `src/assets/css/base.css` のトークンを正とする。
- 写真コンテナは `--radius-media` を使う。情報カード・表はコンテンツに応じた角丸を維持する。

## 導線

- ヘッダー、ハンバーガーメニュー、フッターのナビゲーションは `site.json` だけを編集する。
- 関連ページは3件を基本とし、同一ページへのリンクは置かない。
- PC追従CTAは「お問い合わせ」「見学・ご相談予約」の2本を基本とする。
- アンカーの`id`とサイドナビの`data-target`は、`page.json` の `anchorNavigation.items` から同時に生成する。

## CSSとJavaScript

- `base.css`: トークン、リセット、共通ヘッダー、フッター、ボタン、共通セクション
- `home.css`: トップページ固有として分離したセクション（既存の共通化済みトップページスタイルは、互換性のため`base.css`に維持する）
- `subpages.css`: KV、パンくず、アンカー、左追従ナビ、関連ページ
- その他のCSS: ページまたは機能固有
- JavaScriptはDOM要素の有無を確認してから初期化する。クラスによる状態管理、`requestAnimationFrame`、`IntersectionObserver`を維持する。

## 確認手順

1. `npm run check`
2. `git diff --check`
3. PC・スマホで、メニュー開閉、アンカー遷移、左追従ナビ、追従CTA、フォームの必須項目エラーを確認する。
